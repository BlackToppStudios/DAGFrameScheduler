00340                 /// @brief This is the 1st step (of 6) in a frame.
00341                 /// @details This Iterates over the listing of @ref MonopolyWorkUnit "MonopolyWorkUnit"s and executes each one in the order
00342                 /// it was added. This should be considered as consuming all available CPU time until it returns. This call blocks until execution
00343                 /// of monopolies is complete.

00345 
00346                 /// @brief This is the 2nd step (of 6) in a frame.
00347                 /// @details This starts all the threads on their work. Until @ref JoinAllThreads() is called some thread may still be working.
00348                 /// This thread starts the man @ref algorithm_sec "scheduling algorithm" working on every thread except the calling thread. This
00349                 /// call does not block and tends to return very quickly.
00350                 /// @n @n
00351                 /// This checks the amount of threads as set by @ref SetFrameLength(Whole) "SetFrameLength(Whole)". It creates any
00352                 /// @ref ThreadSpecificStorage instances required and creates threads if they are required. If the build option
00353                 /// @ref MEZZ_USEBARRIERSEACHFRAME Mezz_MinimizeThreadsEachFrame was enabled then this will reuse threads from previous frames,
00354                 /// otherwise this will re-use thread specific resources and create a new set of threads. Re-use of threads is synchronized
00355                 /// with the @ref Barrier StartFrameSync member variable. It is unclear, and likely platform specific, which option has
00356                 /// better performance characteristics.
00357                 /// @warning While this is running any changes to the @ref FrameScheduler be made with an atomic operation like the
00358                 /// @ref AtomicCompareAndSwap32 "AtomicCompareAndSwap32" or @ref AtomicAdd "AtomicAdd". Any other threads
00359                 /// @ref ThreadSpecificStorage or workunit may be accessed as long as they are protected by some kind of synchronization
