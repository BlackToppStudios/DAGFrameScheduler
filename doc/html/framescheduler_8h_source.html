391  /// @details This just calls the following functions in the order presented:
392  /// @code
393  /// void FrameScheduler::DoOneFrame()

404  /// allow maximum integration with existing projects. It can also be used to prevent a giant migration and replace it with a piecemeal upgrade.
405  /// @warning Do not call this on an unsorted set of WorkUnits. Use @ref FrameScheduler::SortWorkUnitsAll() and the @ref DependentGraph to sort WorkUnits after
406  /// they are inserted into the frame scheduler for the first time. This doesn't need to happen each frame, just after any new work units are added or removed
407  /// (except Monopolies) or you want to take the most recent performance number into account.

415  /// @brief This is the 1st step (of 6) in a frame.
416  /// @details This iterates over the listing of @ref MonopolyWorkUnit "MonopolyWorkUnit"s and executes each one in the order
417  /// it was added. This should be considered as consuming all available CPU time until it returns. This call blocks until execution
418  /// of monopolies is complete.
419  virtual void RunAllMonopolies();
420 
421  // If the monopolies need some cleanup, it could be done here. Not much should be done here, most work is better as a work unit.
422 
423  /// @brief This is the 2nd step (of 6) in a frame.
424  /// @details This starts all the threads on their work. Until @ref JoinAllThreads() is called some thread may still be working.
425  /// This thread starts the man @ref algorithm_sec "scheduling algorithm" working on every thread except the calling thread. This
426  /// call does not block and tends to return very quickly.
427  /// @n @n
428  /// This checks the amount of threads as set by @ref SetFrameLength "SetFrameLength". It creates any
429  /// @ref ThreadSpecificStorage instances required and creates threads if they are required. SwapAllBufferedResources() Is called on
430  /// each ThreadSpecificStorage before being passed into the thread. If extra double buffered resources are required per thread
431  /// and they need to be swapped each frame, SwapAllBufferedResources() should be inherited or adjusted to account for these new
432  /// resources.
433  /// @n @n
434  /// If the build option
435  /// @ref MEZZ_USEBARRIERSEACHFRAME Mezz_MinimizeThreadsEachFrame was enabled then this will reuse threads from previous frames,
436  /// otherwise this will re-use thread specific resources and create a new set of threads. Re-use of threads is synchronized
437  /// with the @ref Barrier StartFrameSync member variable. It is unclear, and likely platform specific, which option has
438  /// better performance characteristics.
439  /// @warning While this is running any changes to the @ref FrameScheduler must be made with an atomic operation like the
440  /// @ref AtomicCompareAndSwap32 "AtomicCompareAndSwap32" or @ref AtomicAdd "AtomicAdd". Any other threads
441  /// workunit may be accessed as any normal shared data, but Thread specific Resources should not be accessed while this runs.
442  /// @warning This uses a Spinlock to prevent accesss to ThreadSpecificResources that the LogAggregator needs. This is unlocked
443  /// in RunMainThreadWork.
444  virtual void CreateThreads();
445 
446  // If it must be run on the main thread and must be run first each frame it could go here, but I think you should
447  // just make a work unit out of it and make other work units depend on it. Right now all the other thread are working.
448 
449  /// @brief This is the 3rd step (of 6) in a frame.
450  /// @details This runs the main portion of the @ref algorithm_sec "scheduling algorithm" on the main thread. This call
451  /// blocks until the execution of all workunits with main thread affinity are complete and all other work units have at
452  /// least started. This could return and other threads could still be working.
453  /// @n @n
454  /// Before executing any work this checks a flag to determine if it should log the current work unit dependencies. After
455  /// this check or logging has occurred this then this releases the spinlock on the log, so there is a chance of brief
456  /// contention between a LogAggregator and this if depedencies have just changed or change frequently.
457  /// @warning This uses a Spinlock to prevent accesss to ThreadSpecificResources that the LogAggregator needs. This should
458  /// be called immediately after CreateThreads to minimize any possible contention.
459  virtual void RunMainThreadWork();
460 
461  // Alled the work units have at least been started, and quite possible most have finished. It is possible a few threads
462  // have finished. But if you must do something after all the workunits have started and most finished, while the threads
463  // remain in memory, then this is the place to do it.
464 
465  /// @brief This is the 4th step (of 6) in a frame.
466  /// @details Used when completing the work of a frame, to cleaning end the execution of the threads. This function will
467  /// only return when all the work started by @ref CreateThreads() and @ref RunMainThreadWork() have completed. This call
468  /// blocks until all threads executing. If a thread takes too long then this simply waits for it to finish. No attempt is
469  /// made to timeout or interupt a work unit before it finishes.
470  void JoinAllThreads();
471 
472  // All the threads have been cleaned up or paused. All the work units have finished. This is a reasonable place for heuristics
473  // and maybe other kinds of work. but maybe that stuff could go in a monopoly instead. This might be a good place to check
474  // statuses of work units, but all the interesting statuses kill the program.
475 
476  /// @brief This is the 5th step (of 6) in a frame.
477  /// @details Take any steps required to prepare all owned WorkUnits for execution next frame. This usually includes reseting
478  /// all the work units running state to @ref NotStarted "NotStarted". This can cause work units to be executed multiple times
479  /// if a thread is still executing.
480  virtual void ResetAllWorkUnits();
481 
482  // All the work units are ready for the next frame, but no real waiting has occurred yet.
483 
484  /// @brief This is the final step (of 6) in a frame.

487  /// use. If a frame took too long to execute this calculates that and returns.
488  /// @n @n
489  /// Wait 1/TargetFrame seconds, minus time already run. This also starts the timer for the next frame so
490  /// any other logic that needs to run after the frame does not interfere with frame timing. Because
