00055         /// @brief Stores data about a single work unit so that it can easily be sorted.
00056         class MEZZ_LIB WorkUnitKey
00057         {
00058             public:
00059                 /// @brief The WorkUnit this carries metadata for.
00060                 WorkUnit* Unit;
00061                 
00062                 /// @brief This stores how many things must run after the target workunit.
00063                 /// @details This is most important sorting factor. In the Default algorithm
00064                 /// All of the Workunit that the most workunits depend on are run before the
00065                 /// any others. Since this number is usually low (less than 10ish) in large
00066                 /// groups of workunits, then there must be a tie-breaker. This should be
00067                 /// calculated including the dependers of all the dependers (I don't think
00068                 /// counting a workunit twice hurts, and it may even help if something
00069                 /// actually depend on multiple workunits that depend on this.)
00070                 /// @note I chose the word depender instead of dependent, to make the distinction between this field and dependencies more clear when communicated verbally. In some english dialects dependencies it pronounced Dee-pen-dent-sees only one syllable off from Dee-pen-dent.
00071                 Whole Dependers;
00072 
00073                 /// @brief Some representation of the time the target workunit takes to execute.
00074                 /// @details In the default algorithm this is used to determine what executes
00075                 /// first after dependents are counted. It is important to know what takes the longest time
00076                 /// and start it first so that it might finish coterminously with shorter
00077                 /// workunits. This is used second because length of execution of one workunit
00078                 /// means nothing if many other workunits run serialized in one thread rather than
00079                 /// in parrellel.
00080                 Whole Time;
00081 
00082                 /// @brief Constructor
00083                 /// @param Dependers_ How many items depend on this, This needs to be calculated the same for all WorkUnitKeys
00084                 /// @param Time_ How long is this workunit expected to execute for, This needs to be calculated the same for all WorkUnitKeys
00085                 /// @param WorkUnit_ A pointer to the workunit in question.
00086                 WorkUnitKey(Whole Dependers_, Whole Time_, WorkUnit* WorkUnit_);
00087 
00088                 /// @brief Destructor
00089                 virtual ~WorkUnitKey();
00090 
00091                 /// @brief The function that does the comparison in most containers.
00092                 /// @param rhs The right hand WorkUnitKey when using <.
00093                 /// @details This enforces the algorithm that sorts workunits. The workunit with the most
00094                 /// other work units depending on it is priotized first. Because the longest delays in
00095                 /// execution time can be introduced if WorkUnits are waiting for a dependency and it is
00096                 /// poorly scheduled, this can results in single threaded execution when more could have
00097                 /// been possible. If there is a tie in dependent count, then the work unit that takes
00098                 /// longer to execute is prioritized (this needs to be tested for correctness). This
00099                 /// starts WorkUnits that take longer earlier in the frame, helping to minimize the
00100                 /// length of the critical path. In the event that those are both tied, then the WorkUnit
00101                 /// with the lowest pointer value is priotized to prevent any of WorkUnitKey from comparing
00102                 /// identically.
00103                 /// @return True if the left hand WorkUnit should be prioritized ahead of the right hand work unit.
00104                 virtual bool operator< (const WorkUnitKey& rhs ) const;
